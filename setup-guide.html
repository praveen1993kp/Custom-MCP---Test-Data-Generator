What Is This Project?

The testdata-mcp-server is a Model Context Protocol (MCP) server written in TypeScript. It exposes two tools that AI assistants (like GitHub Copilot or Claude) can call to get test data:

ToolPurposeDatabase Required?
createTestData Generate synthetic fake data using Faker (in memory only) No
getLatestProspects Fetch real prospect leads from MySQL / Opentaps CRM database Yes

Step-by-Step Setup

1 Get the Project Files

Copy or clone the test-data-mcp-server folder to your machine.

# If shared as a zip, unzip it:
unzip test-data-mcp-server.zip
cd test-data-mcp-server

# Or if in a Git repo:
git clone <repo-url>
cd test-data-mcp-server
2 Install Dependencies 
npm install

This installs all required packages:

  • @faker-js/faker — synthetic data generation
  • mysql2 — MySQL database driver
  • dotenv — loads environment variables from .env
  • typescript & ts-node — TypeScript compilation & execution
3 Configure Environment Variables

Create a .env file in the project root (or copy the example):

# .env file

# MySQL Database Configuration
DB_HOST=20.235.243.83
DB_PORT=3306
DB_USER=sandbox
DB_PASSWORD=your_password_here
DB_NAME=opentaps

# Environment
NODE_ENV=development
VariableDescriptionRequired?
DB_HOSTMySQL server hostname or IP addressOnly for getLatestProspects
DB_PORTMySQL port (default: 3306)Optional
DB_USERDatabase usernameOnly for getLatestProspects
DB_PASSWORDDatabase passwordOnly for getLatestProspects
DB_NAMEDatabase schema nameOnly for getLatestProspects
📝 Note If you only plan to use createTestData (fake data generation), you can skip this step entirely — no database is needed.
4 Build the Project

Compile TypeScript source into JavaScript:

npm run build

This runs the TypeScript compiler (tsc) and outputs JavaScript files into the dist/ folder.

5 Run the Server

You have two options:

# Option A: Run the compiled build (production)
npm start

# Option B: Run directly from TypeScript (development)
npm run dev

The server starts listening on stdin for JSON-RPC 2.0 requests and responds on stdout.

6 Verify It Works

Test by sending a JSON-RPC request to list available tools:

# List all tools
echo '{"jsonrpc":"2.0","id":1,"method":"tools/list"}' | npm start

Test the createTestData tool (no database needed):

# Generate 3 fake records with Indian IT company names
echo '{"jsonrpc":"2.0","id":2,"method":"tools/call","params":{"name":"createTestData","arguments":{"count":3,"companyStyle":"indian-it"}}}' | npm start

Test the getLatestProspects tool (requires database):

# Fetch 5 prospects with firstName and companyName
echo '{"jsonrpc":"2.0","id":3,"method":"tools/call","params":{"name":"getLatestProspects","arguments":{"fields":["firstName","companyName"],"limit":5}}}' | npm start

Connecting to an MCP Client (VS Code / Claude Desktop)

The mcp.json file in the project root defines how an MCP client should launch this server:

{
  "name": "testdata-mcp-server",
  "command": "node",
  "args": ["dist/index.js"],
  "env": {
    "DB_HOST": "20.235.243.83",
    "DB_PORT": "3306",
    "DB_NAME": "opentaps",
    "DB_USER": "sandbox",
    "DB_PASSWORD": "your_password_here"
  }
}

For VS Code (GitHub Copilot)

  1. Open VS Code Settings (Cmd + , on Mac / Ctrl + , on Windows)
  2. Search for MCP in settings
  3. Add a new MCP server entry pointing to this project's mcp.json
  4. Restart VS Code — the tools will now appear in Copilot Chat

For Claude Desktop

  1. Open Claude Desktop settings → Developer → MCP Servers
  2. Add a new server with command: node, args: ["dist/index.js"]
  3. Set the working directory to the project folder
  4. Add the environment variables from the table above

Tool Reference

Generates synthetic test data in memory using Faker. No database required.

ParameterTypeRequiredDescription

Troubleshooting

ProblemSolution
node: command not found Install Node.js >= 18 from https://nodejs.org
npm install fails Delete node_modules/ and package-lock.json, then run npm install again
Missing required database environment variables Create or update the .env file with DB_HOST, DB_USER, DB_PASSWORD, DB_NAME
ECONNREFUSED on database Check that the MySQL host is reachable and credentials are correct
Build errors Run npm run build and check the TypeScript errors in the output
Server produces no output The server reads from stdin — pipe a JSON-RPC request into it (see Step 6)
⚠️ Security Reminder Never commit the .env file to a public Git repository. Add .env to your .gitignore file.
